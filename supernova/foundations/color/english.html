Color System

The Color System is designed to express appearance and communicate clear meaning. The colors come in two different color themes with a specific range of colors.

Designers can clone our Figma Library for all of the defined themes from below.

Color Themes

There are two color themes: Light and Dark. Each user interface component supports both themes consistently. The choice of a specific color theme is based on the content, platform, device settings, and user experience conditions, such as day or night time. The light theme is applied as the default color theme.

Structure

The color library offers four global branches with a range of colors used to support various design decisions.

  • Brand: A set of brand related colors
  • Static: Theme independent colors
  • Light: Colors that work within the Light Theme
  • Dark: Colors that work within the Dark Theme

Most of the colors are organized into color range groups. The color ranges derive from a specific Base Color using a methodology of percentual increase or decrease of the color tone towards the dark or the light scale.

Dynamic Colors

Dynamic Colors are presented by the sets of color ranges that undergo a tonal change depending on the applied color theme.

  • Light Theme
  • Dark Theme

Semantic Colors

Some of the colors are assigned meanings and are used to communicate them in a clear and predictable way, creating a specific expectation for users through a component’s appearance. These color should never be used as decoration to avoid creating ambiguous contexts that might result in misguiding users.

When used to support a user’s immediate action, these colors communicate visual feedback and their contextual meaning such as positive, negative and notice.
